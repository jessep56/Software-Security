# Software-Security

Artemis Financial, my client, is a financial services company with a need to secure its software against vulnerabilities that could impact sensitive financial data. The company tasked me with identifying and addressing software security vulnerabilities, focusing on strengthening the application against known risks. When assessing vulnerabilities, I thoroughly examined dependencies using the OWASP dependency-check tool and identified critical and high-severity issues, ensuring that I followed secure coding practices to minimize exposure to threats. Coding securely is crucial, as it protects both the company’s data and its reputation, ultimately contributing to its overall well-being.

A particularly challenging aspect of the vulnerability assessment was accurately identifying and categorizing vulnerabilities without introducing new issues in the refactoring process. I added multiple layers of security by implementing a secure encryption cipher, self-signed SSL certificates, and checksum verification, which enhanced data integrity and secure communication. To verify functionality and security post-refactoring, I ran extensive tests, including secondary static testing, and carefully reviewed the dependency-check report for new vulnerabilities.

In this project, tools like Maven and best practices in secure coding were invaluable, as they not only streamlined the process but also reinforced my approach to safeguarding the codebase. For future assessments, I would rely on industry-standard tools like dependency-check and consider adding automated static analysis to monitor changes continuously. If presenting this work to potential employers, I would showcase my ability to follow secure coding practices, perform thorough vulnerability assessments, and apply layered security enhancements to meet client requirements. This assignment demonstrates my skills in identifying risks, implementing mitigations, and validating code security and functionality.
